# CYB-006 — API Abuse & Token Misuse

## Rule Information

Rule ID: CYB-006

Domain: Cyber Resilience

Category: API Security / Token Exploitation

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under API abuse conditions, including token misuse, excessive requests, and unauthorized API operations, and assess cascading impacts across services and infrastructure.

---

## Scenario

API credentials or tokens are misused through theft, leakage, over-permissioning, or abuse of exposed endpoints. This results in unauthorized access, data extraction, or service degradation due to excessive API consumption.

The scenario evaluates:

- API authentication integrity
- Token validity and lifecycle control
- Rate limiting effectiveness
- Endpoint abuse detection
- Service dependency stability

---

## Primary Indicators

- Unusual API request volume
- Repeated authentication failures
- Token reuse across regions or devices
- Endpoint overload patterns
- Spike in 4xx/5xx responses
- Abnormal data extraction volume
- Rate limit trigger frequency

---

## Risk Levels

### GREEN

API activity within normal usage patterns.

Authentication and authorization functioning correctly.

No contingency actions required.

---

### YELLOW

Early signs of abnormal API usage.

Monitor:

- Increased request rates
- Minor authentication anomalies
- Elevated error rates

Action:

Increase monitoring and validate API key security.

---

### ORANGE

Active API abuse detected.

Indicators:

- Excessive request volume
- Unauthorized endpoint access
- Token misuse confirmed
- Service degradation begins

Potential Cascade:

API Abuse

↓

Service Overload

↓

Performance Degradation

↓

Partial Outage

Action:

Revoke compromised tokens, enforce stricter rate limits, and isolate affected services.

---

### RED

Critical API exploitation event.

Indicators:

- System-wide API saturation
- Large-scale data exposure
- Authentication system bypass
- Service outage

Potential Cascade:

API Exploitation

↓

Infrastructure Stress

↓

System Failure

↓

Cross-Domain Systemic Risk

Action:

Shut down compromised API endpoints, rotate all credentials, and activate emergency incident response.

---

## Cross-Domain Dependencies

Cyber (CYB)

↓

Data Centre (DC)

↓

Infrastructure (INF)

↓

API misuse may impact all dependent systems including financial transactions and operational services.

---

## Contingency Actions

- Revoke and rotate API tokens immediately
- Enforce strict rate limiting
- Monitor endpoint anomalies continuously
- Disable compromised integrations
- Audit API access logs
- Escalate to security operations team
