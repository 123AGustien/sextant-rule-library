# CYB-007 — Supply Chain Cyber Compromise

## Rule Information

Rule ID: CYB-007

Domain: Cyber Resilience

Category: Third-Party / Supply Chain Risk

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under supply chain cyber compromise conditions, where a trusted third-party vendor, library, or dependency is compromised and introduces downstream risk into the primary system.

---

## Scenario

A trusted external dependency (software vendor, update pipeline, API provider, or library) is compromised. Malicious code, vulnerabilities, or backdoors propagate into the primary environment through legitimate update or integration channels.

The scenario evaluates:

- Third-party trust integrity
- Dependency risk exposure
- Software supply chain security
- Update pipeline validation
- Cross-system contamination potential

---

## Primary Indicators

- Unexpected dependency updates
- Integrity check failures
- Suspicious vendor behavior
- Code signature mismatches
- Unverified package installations
- CI/CD pipeline anomalies
- External API behavior changes

---

## Risk Levels

### GREEN

All third-party dependencies verified and trusted.

No anomalies detected in supply chain.

No contingency actions required.

---

### YELLOW

Suspicious activity in external dependencies.

Monitor:

- Unverified updates
- Minor integrity mismatches
- Vendor communication anomalies

Action:

Increase dependency validation and review third-party changes.

---

### ORANGE

Confirmed supply chain compromise risk.

Indicators:

- Malicious or altered dependency detected
- Unauthorized code execution via update channel
- Partial system compromise through external source
- Integrity validation failures

Potential Cascade:

Supply Chain Compromise

↓

System Infiltration

↓

Service Degradation

↓

Infrastructure Risk

Action:

Isolate affected dependencies, roll back to trusted versions, and disable automated updates.

---

### RED

Critical supply chain attack confirmed.

Indicators:

- Widespread system compromise via trusted vendor
- Multiple systems infected through update pipeline
- Severe operational disruption
- Loss of dependency trust integrity

Potential Cascade:

Supply Chain Attack

↓

System-Wide Compromise

↓

Infrastructure Failure

↓

Cross-Domain Systemic Crisis

Action:

Immediately disable affected supply chain channels, perform full system audit, and restore from verified clean builds.

---

## Cross-Domain Dependencies

Cyber (CYB)

↓

Data Centre (DC)

↓

Infrastructure (INF)

↓

Financial (FIN)

Supply chain compromise may propagate across all domains due to shared dependencies and trusted integrations.

---

## Contingency Actions

- Disable affected third-party dependencies immediately
- Roll back to verified stable versions
- Audit all external integrations
- Validate code and package integrity
- Monitor CI/CD pipelines continuously
- Escalate to security governance team
