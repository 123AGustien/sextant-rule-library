# CYB-002 — Data Breach & Credential Leak

## Rule Information

Rule ID: CYB-002

Domain: Cyber Resilience

Category: Data Breach / Identity Compromise

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under conditions of data breach and credential leakage, and assess cascading impacts across identity systems, infrastructure access, financial exposure, and operational trust.

---

## Scenario

Sensitive data is exposed due to misconfiguration, phishing, insider threat, API vulnerability, or external breach. Credentials, personal data, or system keys are leaked, enabling unauthorized access and downstream exploitation.

The scenario evaluates:

- Credential exposure impact
- Identity system compromise
- Unauthorized access escalation
- Data integrity and confidentiality loss
- Cross-domain trust degradation

---

## Primary Indicators

- Number of leaked credentials
- Severity of exposed data
- Unauthorized login attempts
- Privilege escalation events
- API key misuse detection
- Data exfiltration volume
- Identity verification failures

---

## Risk Levels

### GREEN

No evidence of data breach or credential exposure.

Identity systems fully secure.

No contingency actions required.

---

### YELLOW

Suspicious access patterns or minor exposure risk.

Monitor:

- Unusual authentication attempts
- Weak credential usage
- API anomaly detection

Action:

Increase authentication monitoring and enforce security policy checks.

---

### ORANGE

Confirmed partial data breach or credential exposure.

Indicators:

- Exposed credentials detected
- Unauthorized access attempts successful
- Limited data exfiltration
- Elevated privilege misuse risk

Potential Cascade:

Credential Leak

↓

Unauthorized Access

↓

System Manipulation

↓

Service Integrity Degradation

Action:

Revoke compromised credentials, enforce resets, and isolate affected systems.

---

### RED

Severe data breach with widespread credential compromise.

Indicators:

- Large-scale credential exposure
- System-wide unauthorized access risk
- Critical data exfiltration
- Identity system compromise

Potential Cascade:

Data Breach

↓

System Takeover Risk

↓

Infrastructure Exploitation

↓

Financial & Operational Impact

↓

Cross-Domain Systemic Risk

Action:

Immediate credential revocation, full security lockdown, forensic investigation, and system recovery protocols.

---

## Cross-Domain Dependencies

Cyber (CYB)

↓

Data Centre (DC)

↓

Infrastructure (INF)

↓

Financial (FIN)

Credential compromise may enable cascading failures across all connected domains.

---

## Contingency Actions

- Revoke compromised credentials immediately
- Force system-wide credential rotation
- Monitor authentication systems continuously
- Isolate affected identity providers
- Audit all privileged access logs
- Escalate to incident response team
