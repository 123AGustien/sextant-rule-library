# CYB-008 — Cloud Misconfiguration Exposure

## Rule Information

Rule ID: CYB-008

Domain: Cyber Resilience

Category: Cloud Security / Configuration Risk

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under cloud misconfiguration scenarios, where incorrect access controls, exposed storage, or insecure configurations lead to unintended public exposure or unauthorized access.

---

## Scenario

A cloud environment is misconfigured due to human error, automation failure, or policy drift. Sensitive resources such as databases, storage buckets, APIs, or virtual machines become exposed to unauthorized access.

The scenario evaluates:

- Access control integrity
- Public exposure detection
- Configuration drift impact
- Data leakage risk
- Cloud security posture management

---

## Primary Indicators

- Publicly accessible storage or endpoints
- Unrestricted IAM permissions
- Unexpected external traffic to cloud resources
- Missing or weak authentication controls
- Configuration drift alerts
- Security group misalignment
- Data access anomalies

---

## Risk Levels

### GREEN

Cloud infrastructure securely configured.

No exposure detected.

No contingency actions required.

---

### YELLOW

Minor configuration drift detected.

Monitor:

- Non-critical permission expansions
- Weak security group settings
- Early exposure warnings

Action:

Review and correct configuration drift.

---

### ORANGE

Significant cloud misconfiguration exposure.

Indicators:

- Public exposure of sensitive resources
- Over-permissive IAM roles
- Unauthorized data access risk
- Security policy violations

Potential Cascade:

Cloud Exposure

↓

Data Leakage

↓

Service Integrity Risk

↓

Operational Impact

Action:

Restrict access immediately, correct configurations, and audit exposed resources.

---

### RED

Critical cloud exposure event.

Indicators:

- Large-scale public data exposure
- Sensitive system access compromise
- Severe security control failure
- Active exploitation of exposed resources

Potential Cascade:

Cloud Misconfiguration

↓

Mass Data Exposure

↓

System Compromise

↓

Cross-Domain Systemic Risk

Action:

Immediately lock down cloud environment, revoke access keys, and perform full security audit and remediation.

---

## Cross-Domain Dependencies

Cyber (CYB)

↓

Data Centre (DC)

↓

Infrastructure (INF)

↓

Financial (FIN)

Cloud exposure can rapidly escalate into financial and infrastructure-level risk.

---

## Contingency Actions

- Immediately restrict public access to affected resources
- Audit IAM roles and permissions
- Correct misconfigured security groups
- Rotate exposed credentials
- Enable enhanced monitoring and logging
- Escalate to cloud security operations team
