# INF-002 — DNS Infrastructure Failure

## Rule Information

Rule ID: INF-002

Domain: Infrastructure Resilience

Category: Domain Name System (DNS) Failure

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under DNS infrastructure failure, where domain resolution becomes degraded or unavailable, impacting service accessibility even when core systems remain operational.

---

## Scenario

A DNS failure occurs due to misconfiguration, provider outage, propagation delay, poisoning, or recursive resolver instability. Users are unable to resolve domain names to IP addresses, causing widespread service accessibility issues.

The scenario evaluates:

- DNS resolution reliability
- Recursive resolver stability
- Authoritative server availability
- Cache integrity
- Service discoverability under failure

---

## Primary Indicators

- Domain resolution failures
- NXDOMAIN or SERVFAIL spikes
- Increased DNS query latency
- Resolver timeouts
- Inconsistent record propagation
- Authoritative server unavailability
- Cache poisoning anomalies

---

## Risk Levels

### GREEN

DNS resolution fully operational.

All domains resolving correctly.

No contingency actions required.

---

### YELLOW

Minor DNS instability detected.

Monitor:

- Increased query latency
- Intermittent resolution delays
- Resolver load increase

Action:

Monitor DNS health and verify redundancy across providers.

---

### ORANGE

Significant DNS degradation.

Indicators:

- Frequent resolution failures
- Partial domain unavailability
- Resolver instability
- Delayed propagation of DNS records

Potential Cascade:

DNS Instability

↓

Service Access Disruption

↓

Application Reachability Issues

↓

Operational Impact

Action:

Switch to backup DNS providers and flush/repair caching layers.

---

### RED

Critical DNS infrastructure failure.

Indicators:

- Widespread domain resolution failure
- Complete service inaccessibility via domain names
- Authoritative server outage
- Resolver network collapse

Potential Cascade:

DNS Failure

↓

Global Service Inaccessibility

↓

Infrastructure Isolation

↓

Cross-Domain Systemic Risk

Action:

Activate emergency DNS failover, restore authoritative services, and reroute traffic using direct IP or backup naming systems.

---

## Cross-Domain Dependencies

Infrastructure (INF)

↓

Data Centre (DC)

↓

Cyber (CYB)

↓

Financial (FIN)

DNS failure can make fully functional systems appear offline, creating systemic operational disruption.

---

## Contingency Actions

- Switch to redundant DNS providers immediately
- Flush and rebuild DNS caches
- Validate authoritative zone integrity
- Monitor resolver health globally
- Enable direct IP fallback routing if required
- Escalate to network operations team
