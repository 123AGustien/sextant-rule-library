# INF-006 — Cloud Region Outage

## Rule Information

Rule ID: INF-006

Domain: Infrastructure Resilience

Category: Cloud Infrastructure Failure

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under cloud region outage conditions where an entire cloud region becomes unavailable, impacting hosted services, workloads, and dependent systems.

---

## Scenario

A full cloud region experiences outage due to infrastructure failure, control plane disruption, natural disaster, or provider-level incident. All services hosted in the region become partially or fully unavailable.

The scenario evaluates:

- Regional service availability
- Failover to secondary regions
- Cloud control plane resilience
- Data redundancy effectiveness
- Multi-region architecture stability

---

## Primary Indicators

- Regional service downtime
- Cloud API failures
- Compute instance unavailability
- Storage access disruption
- Load balancer failover triggers
- Control plane errors
- Multi-region sync delays

---

## Risk Levels

### GREEN

Cloud region operating normally.

All services fully available.

No contingency actions required.

---

### YELLOW

Minor regional instability.

Monitor:

- Increased latency in region
- Intermittent service disruptions
- Control plane delays

Action:

Verify multi-region redundancy readiness.

---

### ORANGE

Significant regional degradation.

Indicators:

- Partial region service loss
- Compute or storage instability
- Delayed failover activation
- Reduced service availability

Potential Cascade:

Region Instability

↓

Service Degradation

↓

Failover Stress

↓

Operational Impact

Action:

Initiate controlled failover to secondary region.

---

### RED

Complete cloud region outage.

Indicators:

- Full regional service unavailability
- Control plane failure
- Loss of compute and storage access
- Multi-service disruption

Potential Cascade:

Region Failure

↓

Service Collapse

↓

Cross-Region Failover Stress

↓

Cross-Domain Systemic Risk

Action:

Activate disaster recovery region, reroute all workloads, and restore services from backup infrastructure.

---

## Cross-Domain Dependencies

Infrastructure (INF)

↓

Data Centre (DC)

↓

Cyber (CYB)

↓

Financial (FIN)

Cloud region outages propagate across all dependent digital systems and financial operations.

---

## Contingency Actions

- Trigger multi-region failover
- Redirect traffic to backup regions
- Validate data replication integrity
- Restore cloud services via DR environment
- Monitor control plane recovery
- Escalate to cloud provider support
