# INF-008 — Certificate Authority / TLS Failure

## Rule Information

Rule ID: INF-008

Domain: Infrastructure Resilience

Category: Security Infrastructure Failure (TLS / PKI)

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under TLS/SSL and certificate authority (CA) failures affecting encrypted communication, trust validation, and secure service access.

---

## Scenario

Certificate validation systems fail due to expired certificates, CA outage, misconfiguration, or revocation issues. Secure connections cannot be established, causing service trust failures and access disruption.

The scenario evaluates:

- TLS handshake success rate
- Certificate validity checks
- Trust chain integrity
- Secure API communication
- Browser/service access compatibility

---

## Primary Indicators

- Certificate expiration errors
- TLS handshake failures
- Revocation list errors (CRL/OCSP)
- Increased HTTPS connection failures
- API authentication failures due to TLS
- Trust chain validation errors
- Secure service downtime

---

## Risk Levels

### GREEN

All certificates valid and trusted.

Secure communication functioning normally.

No contingency actions required.

---

### YELLOW

Minor certificate or TLS anomalies detected.

Monitor:

- Approaching certificate expiry
- Intermittent handshake failures
- OCSP/CRL latency

Action:

Renew certificates and validate trust chains.

---

### ORANGE

Significant TLS/PKI degradation.

Indicators:

- Multiple certificate validation failures
- Partial service inaccessibility
- Broken trust chains
- API communication disruption

Potential Cascade:

TLS Failure

↓

Secure Connection Breakdown

↓

Service Access Failure

↓

Operational Impact

Action:

Rotate certificates, restore trust chains, and reconfigure TLS endpoints.

---

### RED

Critical PKI infrastructure failure.

Indicators:

- Widespread HTTPS failure
- Complete trust chain breakdown
- System-wide secure access loss
- Authentication service disruption

Potential Cascade:

PKI Failure

↓

Secure Infrastructure Collapse

↓

Service Outage

↓

Cross-Domain Systemic Risk

Action:

Emergency certificate reissuance, restore CA services, and re-establish trust infrastructure immediately.

---

## Contingency Actions

- Renew and redeploy certificates immediately
- Restore CA services or backups
- Validate trust chains across all systems
- Monitor TLS handshake logs
- Reconfigure affected endpoints
- Escalate to security infrastructure team
