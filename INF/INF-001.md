# INF-001 — Regional Network Outage

## Rule Information

Rule ID: INF-001

Domain: Infrastructure Resilience

Category: Network Connectivity Failure

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate system resilience under regional or large-scale network outage conditions affecting connectivity between users, data centres, and external services, and assess cascading impacts across all dependent domains.

---

## Scenario

A regional network disruption occurs due to ISP failure, submarine cable damage, routing instability, DNS failure, or large-scale infrastructure degradation. Connectivity between systems becomes partially or fully unavailable.

The scenario evaluates:

- Network reachability
- External connectivity stability
- DNS resolution integrity
- Inter-region communication
- Service accessibility degradation

---

## Primary Indicators

- Packet loss across regions
- DNS resolution failures
- Increased latency or timeouts
- Inter-data-centre connectivity loss
- BGP routing instability
- ISP backbone degradation
- Service endpoint unavailability

---

## Risk Levels

### GREEN

Full network connectivity is stable.

All services reachable across regions.

No contingency actions required.

---

### YELLOW

Early signs of network instability.

Monitor:

- Increased latency
- Intermittent packet loss
- DNS delays

Action:

Increase monitoring of routing and connectivity layers.

---

### ORANGE

Significant regional network degradation.

Indicators:

- Partial regional outages
- High latency and packet loss
- DNS resolution instability
- Service accessibility disruption

Potential Cascade:

Network Instability

↓

Service Degradation

↓

Inter-System Communication Failure

↓

Operational Impact

Action:

Activate redundant routing paths and reroute traffic through stable regions.

---

### RED

Critical regional network outage.

Indicators:

- Complete loss of regional connectivity
- Severe service unavailability
- Routing system collapse
- Cross-region communication failure

Potential Cascade:

Network Outage

↓

Infrastructure Isolation

↓

Service Disruption

↓

Cross-Domain Systemic Risk

Action:

Activate emergency rerouting, failover to backup regions, and restore connectivity via alternative backbone providers.

---

## Cross-Domain Dependencies

Infrastructure (INF)

↓

Data Centre (DC)

↓

Cyber (CYB)

↓

Financial (FIN)

Network outages can cascade into system-wide operational disruption affecting all domains.

---

## Contingency Actions

- Activate redundant network paths immediately
- Failover to secondary regions
- Monitor routing table stability
- Validate DNS integrity
- Coordinate with ISP providers
- Escalate if outage persists or expands
