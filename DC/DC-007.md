# DC-007 — Network Hardware Failure

## Rule Information

Rule ID: DC-007

Domain: Data Centre Resilience

Category: Network Infrastructure Failure

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate data centre resilience during physical or logical failure of core network hardware and assess cascading impacts across compute, service availability, and connected domains.

---

## Scenario

A critical network device (router, switch, or core interconnect) fails due to hardware malfunction, firmware corruption, overheating, or configuration failure. This leads to partial or full network segmentation within the data centre.

The scenario evaluates:

- Core network redundancy
- Routing stability
- Failover mechanisms
- Service connectivity
- Internal traffic distribution

---

## Primary Indicators

- Core switch status
- Router availability
- Link failure rates
- Routing table stability
- Packet drop rate
- Network segmentation events
- Failover activation time

---

## Risk Levels

### GREEN

All network hardware operating normally.

Full redundancy is active and routing is stable.

No contingency actions required.

---

### YELLOW

Early signs of network hardware instability.

Monitor:

- Link errors
- Device CPU/memory load
- Packet retransmissions
- Redundancy status

Action:

Increase monitoring and verify failover readiness.

---

### ORANGE

Partial network hardware degradation.

Indicators:

- Intermittent link failures
- Reduced redundancy capacity
- Packet loss increase
- Routing instability

Potential Cascade:

Hardware Degradation

↓

Network Instability

↓

Service Latency

↓

Application Degradation

Action:

Activate failover systems, reroute traffic, and isolate failing hardware.

---

### RED

Critical network hardware failure.

Indicators:

- Core switch/router outage
- Network segmentation
- Loss of redundancy
- Severe packet loss

Potential Cascade:

Network Failure

↓

Service Outage

↓

Infrastructure Disruption

↓

Cross-Domain Systemic Risk

Action:

Initiate emergency network failover, restore connectivity via backup paths, and activate disaster recovery if required.

---

## Cross-Domain Dependencies

Data Centre (DC)

↓

Infrastructure (INF)

↓

Cyber (CYB)

↓

Network hardware failure may disrupt digital services, monitoring systems, and financial operations dependent on connectivity.

---

## Contingency Actions

- Activate redundant network paths
- Isolate failed network devices
- Restore routing stability
- Prioritise critical service traffic
- Monitor segmentation risks
- Escalate if failover fails or instability persists
