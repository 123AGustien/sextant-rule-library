# DC-008 — Storage System Degradation

## Rule Information

Rule ID: DC-008

Domain: Data Centre Resilience

Category: Data Storage / IOPS Stress

Status: Active

Version: 1.0

Repository: Sextant Rule Library

---

## Purpose

Evaluate data centre resilience under storage system degradation, including reduced I/O performance, disk failures, and storage latency escalation, and assess cascading effects across compute, applications, and dependent domains.

---

## Scenario

Storage infrastructure begins to degrade due to hardware wear, controller failure, capacity exhaustion, or latency spikes. This results in reduced read/write performance and increased risk of data unavailability or corruption.

The scenario evaluates:

- Storage IOPS performance
- Latency trends
- Disk health status
- RAID/replication integrity
- Cache efficiency
- Data throughput stability

---

## Primary Indicators

- Disk failure rate
- Storage latency (read/write)
- IOPS saturation
- RAID degradation status
- Replication lag
- Controller health
- Queue depth

---

## Risk Levels

### GREEN

Storage systems operating normally.

Performance and redundancy are within safe thresholds.

No contingency actions required.

---

### YELLOW

Early signs of storage stress.

Monitor:

- Latency increase
- Disk health warnings
- Queue depth
- Replication lag

Action:

Increase monitoring and prepare storage optimization.

---

### ORANGE

Significant storage degradation.

Indicators:

- Elevated latency
- Reduced IOPS capacity
- Degraded RAID groups
- Increasing disk errors

Potential Cascade:

Storage Degradation

↓

Application Slowdown

↓

Data Access Delays

↓

Service Performance Impact

Action:

Initiate storage balancing, replace failing components, and reduce non-critical workloads.

---

### RED

Critical storage system failure.

Indicators:

- Multiple disk failures
- Storage array outage
- Data unavailability
- High corruption risk

Potential Cascade:

Storage Failure

↓

Data Loss Risk

↓

Application Outage

↓

Infrastructure Stress

↓

Cross-Domain Systemic Risk

Action:

Activate disaster recovery, restore from backups, and isolate failing storage systems immediately.

---

## Cross-Domain Dependencies

Data Centre (DC)

↓

Infrastructure (INF)

↓

Cyber (CYB)

↓

Financial (FIN)

Storage degradation may impact operational continuity, cybersecurity monitoring, and financial transaction systems.

---

## Contingency Actions

- Monitor storage health continuously
- Replace failing disks proactively
- Balance I/O workloads
- Validate data integrity regularly
- Maintain backup and replication systems
- Escalate if degradation accelerates
